New in Version 6.0.1

This is a bug fix release. All changes are described below.

This version of SasView is built with Sasmodels v1.0.9, Sasdata v0.9.0, and bumps v0.9.3.

What's Changed

General GUI Fixes

  • Open a pop-up on start-up if a new version is available by @lucas-wilkins in #3216
  • Removed superfluous signal connection causing duplicate theories by @rozyczko in #3199
  • Removing data from the context menu now removes it from perspectives by @rozyczko in #3236

Plotting Fixes

  • Fix to ensure default plot scaling matches the plot type by @krzywon in #3184
  • Delete intermediate theory plots by tab id not model id. by @jamescrake-merani in #3160
  • Linear fit no longer causes graph rescaling by @krzywon in #3187
  • Fix for errors thrown when modifying tick labels by @jamescrake-merani in #3191

Fitting and Other Perspective Fixes

  • Status bar message does not clear on gsc exit by @rozyczko in #3185
  • Guard against bad fit results to ensure fit tabs can run new fits by @rozyczko in #3172
  • A number of model editor bug fixes by @tsole0 in #2901
  • Fix model documentation not showing when plugin folder is empty by @krzywon in #3237
  • Added OpenCL support for the Flatpak release by @jamescrake-merani in #3160

Other Fixes

  • Call the system default csv viewer to allow batch fit results output for any OS by @summerhenson in #3186
  • Remove a button that did nothing by @jamescrake-merani in #3217

Infrastructure Fixes

  • Fixes to the flatpak metadata by @jamescrake-merani in #3150
  • Replace the expired OSX Notarization key by @krzywon in #3183
  • Include new contributors in the contributor file by @krzywon in #3192
  • Fix for sasmodel doc build by @krzywon in #3212

New Contributors

  • @summerhenson made their first contribution in #3186
  • @olihammond made their first contribution by reviewing #3186
